File

  • 传入路径时可以是绝对路径或相对路径，相对路径以根目录开头

  • Windows平台使用 \ 作为路径分隔符，即使用 \\ 表示 一个 \ ; Linux平台使用 / 作为路径分隔符

  • getPath()返回构造方法传入的路径；getAbsolutePath()返回绝对路径；getCanonicalPath()返回规范路径

文件和目录

​ File对象既可以表示文件，也可以表示目录，构造File对象时不会对磁盘进行操作，只有调用方法时才会对磁盘进行操作。

  1. isFile(), isDirectory()：判断是否是存在的文件，目录
  2. 一个文件对象，还有canRead(), canWrite(), canExecute(), length()等方法，表示是否可读，是否可写，是否可执行，文件字节大小

创建和删除文件

​ 当File对象表示文件时，使用createNewFile()**创建一个新文件，delete()删除文件。如果要创建临时文件使用createTempFile()，deleteOnExit()**在JVM退出时删除该文件。

遍历文件和目录

​ 当File对象表示目录时，可以使用list()和listFiles()列出目录下的文件和子目录:

  • boolean mkdir()：创建当前File对象表示的目录
  • boolean mkdirs()：创建当前File对象表示的目录，并在必要时将不存在的父目录创建
  • boolean delete()：删除当前File对象表示的目录，必须为空才能删除成功

InputStream

​ InputStream是所有输入流的抽象类，最重要的方法是int read()。该方法会读取输入流的下一个字节，并返回字节表示的int值，读到末尾返回-1

  • 使用输入流读取数据后，最后要使用close()关闭数据流
  • 读取数据时可能会出现文件不存在等情况，因此需要抛出IOException。为了在出错时可以释放资源，使用try…finally语句，或者try(resource)

缓冲

​ 利用缓冲区一次读取多个字节数据，int read(byte[])，读取若干字节并填充到byte[]数组，返回读取的字节数；int read(byte[],int off, int len)，指定byte[]数组的偏移量和最大填充数

  • 使用read方法进行数据读取时，是阻塞的，即数据读取不完成就无法继续执行代码

String

​ 字符串String内部是通过一个char[]数组表示的，java中“…”代表字符串，’.’代表字符char。

  • java字符串不可变，因为其内部字段为private final char[]以及没有任何修改char[]的方法实现的

  • 比较String是否相同时要使用equals()方法而不能使用==

  • java编译器在初期会将所有的字符串放进常量池，相同的字符串合并成一个对象

String 方法

  1. **equalsIgnoreCase()**：比较两个字符串，忽略大小写

  2. contains(CharSequence):判断字符串是否包含指定字符串

  3. **indexof()，lastIndexOf()**：获得指定子串的索引

  4. **startsWith()，endsWith()**：判断字符串是否以指定子串开头，结尾

  5. **subString()**：获得指定索引的子串

  6. **trim()**：去除字符串首尾空白字符（空格，\t，\r，\n），返回新的字符串

  7. **strip()**：去除字符串首尾空白字符，中文的空格也会被移除

  8. **isEmpty()**：判断字符串是否为空

  9. **isBlank()**：判断字符串是否为空白字符串

  10. **replace()**：替换字符串中的字符

  11. **replaceAll()**：通过正则表达式替换所有符合规则的子串

  12. **split()**：分割字符串，传入正则表达式

  13. **join()**：使用指定字符串连接字符串数组

  14. format()静态方法，formatted()方法：格式化字符串，传入其他参数替换占位符，生成新的字符串

    • %s：显示字符串
    • %d：显示整数
    • %x：显示十六进制整数
    • %f：显示浮点数

  15. **String.valueOf()**：将其他类型数据转化为String

  16. **Boolean.parseBoolean()**：将字符串转为布尔值，转为其他类型类似

  17. **toCharArray()**：将字符串转换为char[]

  • String 和char在内存中总是以Unicode编码
  • 将byte[]转换为String已知编码可以直接转换，转换为byte[]时优先考虑UTF-8编码

包装类

​ 对于八大基本类型定义的包装类：Boolean,Byte,Short,Integer,Long,Float,Double,Character。

包装类的创建

自动装箱，拆箱

​ java编译器会自动在int和integer之间转换

  • 装箱和拆箱发生在编译阶段，会影响代码的执行效率

不变类

​ 所有包装类型都是不变类，即不发生改变，比较包装类时要使用equals()

枚举类

​ 编译器可以自动检测某个值在枚举的集合内，并且不同用途的枚举需要不同的类型来标记

  • enum常量本身带有类型信息，编译器会自动检测类型错误

  • 不可能引用非枚举的值，无法通过编译

  • enum类型的每个常量在JVM中只有唯一一个实例，可以直接用==进行比较，不必一定使用equals()

  • 定义的enum类型无法被继承，无法使用new操作符创建enum，可以将enum用于switch语句

  • 每一个枚举的值都是class的实例，且是唯一的实例，他们有一些方法：

    1. name():返回常量名，无法被覆写
    2. ordinal():返回定义的常量顺序
    3. toString():返回和name()一样的字符串，但可以被覆写

记录类

  • 不变类具有一些特点：定义class时使用final，无法派生子类；每个字段使用final，保证创建实例后无法修改任何字段
  • 由于不变类的编写较为复杂，从java 14开始引入Record类，可以一行写出不变类。和enum类似，要通过record关键字由编译器实现继承

BigInteger

​ java.math.BigInteger可以表示任意大小的整数，BigInteger内部使用int[]数组来模拟非常大的整数。

​ BigInteger进行计算时必须使用实例方法，而且计算速度较慢。BigInteger可以转化为基本类型，如：byteValue(),longValue()等，超出范围会丢失高位信息。如果需要精确转换，需要使用byteValueExact()等方法。

BigDecimal

​ BigDecimal可以表示一个任意大小且精度完全准确的浮点数，是通过一个BigInteger和scale来表示的。使用scale()**表示小数位数，使用stripTrailingZeros()**可以将BigDecimal格式化为一个相等的去掉末尾0的BigDecimal。

  • 如果scale()为负数，则表示BigDecimal为整数，末尾有0
  • 可以使用**setScale()**对BigDecimal设置精度，可以四舍五入也可以截断
  • 进行加减乘时精度不会丢失，但在进行除法时，会有无法除尽的情况，因此需要指定精度和如何截断
  • divideAndRemainder():返回一个容量为2的数组，分别代表商和余数

比较BigDecimal

  • equals()：要求两个的scale()相等
  • 使用compareTo()：返回负数，正数和0，代表小于，大于和等于

集合

​ 可以在内部持有若干java对象，并对外提供访问接口的对象称为集合。java.util包提供了集合类：Collection，它是除Map外所有其他集合类的根接口，java.util主要提供了一下三种类型集合：

  1. List：一种有序列表的集合
  2. Set：一种保证没有重复元素的集合
  3. Map：一种通过键值(key-value)查找的映射表集合

​ java集合设计的特点：一是实现了接口与实现类的分离；二是支持泛型，可以限制在集合中只能放入同一种数据类型的元素。

​ java集合的访问使用迭代器（Iterator）实现，优点是无需知道集合内部的元素是如何储存的

equals方法

​ equals()方法必须满足的条件：

  1. 自反性：对于非null的x来说，x.equals(x)必须返回true
  2. 对称性：对于非null的x和y，如果x.equals(y)为true，那么y.equals(x)也为true
  3. 传递性：对于非null的x,y,z来说，如果x.equals(y)，y.equals(z)，那么x.equals(z)也为true
  4. 一致性：对于非null的x和y来说，只要x和y的状态不变，则x.equals(y)的值总不变
  5. 对于null的比较：x.equals(null)永远为false

  • 对于基本类型的比较使用==，对于引用类型的比较使用equals()

  • 对于基本类型==比较的是他们的值，对于引用类型==比较的是他们在内存中的储存地址

equals方法的编写

  1. 确定相等的定义，即那些属性相同就认为对象相等
  2. 使用instanceof判断是不是相同的对象
  3. 对引用类型使用Objects.equals()比较，对基本类型使用==比较
  • List的contains()和indexof()方法都需要调用equals方法

Map

  2. V get(K key):通过key获得相应的value。key不存在就返回null
  3. boolean containsKey(K key):查询某个key是否存在
  • Map中可以重复放入key-value，但一个key只能关联一个value。Map中key 不能重复，但value可以重复

hashCode

  • Map中作为key的类要正确覆写equals()方法和hashCode()方法
  • HashMap将value值存储在一个大型的数组中，每个value值存储的位置通过对key值进行计算得到，计算得到的值也就是hashcode
  • 两个对象相等，他们对应的hashcode必须相等；两个对象不等，他们对应的

hashcode尽量不等（提高查找效率）

编写hashCode()

​ 两个equals的对象对应的hashcode也必须相同，因此需要覆写相应的hashCode()

  • equals()中用于比较的字段都必须放在hashCode()中用于计算，没有比较的字段绝不可以放在hashCode()中计算
  • hashCode()的编写可以借助Objects.hash()编写

HashMap

​ HashMap默认初始大小是16的数组，经过key.hashCode() & 0xf获得value的索引。超过16个存储空间时会进行扩容到32，依次翻倍。HashMap的存储大小一直是2的n次方。具体的HashMap知识可以查看相应的数据结构

EnumMap

​ key的对象是enum类型，EnumMap可以直接定位到内部数组的索引，不需要使用hashCode()，也不会额外浪费空间

TreeMap

​ HashMap不会对内部的key进行排序，而SortedMap接口定义了对内部key进行排序的Map，它的实现类是TreeMap，这里的排序并不是按照输入的顺序，而是其他的顺序，例如字母排列顺序。

  • TreeMap中作为key的接口必须实现Comparable接口，如果没有实现该接口。必须在创建TreeMap时指定一个自定义排序算法。
  • 排序算法必须严格按照规范编写，即小于返回-1，等于返回0，大于返回1，否则其他接口可能调用出现问题。

Properties

​ Properties用来读取配置文件，它的内部本质上是Hashtable。java默认的配置文件以.properties为扩展名，每行以key=value表示。

Set

​ 存储不重复的key：

  1. boolean add(E e):将元素添加进Set
  2. boolean remove(Object e):将元素从Set中删除
  3. boolean contains(Object e):判断是否包含元素
  • 放入Set的元素也需要正确实现equals()和hashCode()方法

HashSet

​ HashSet其实是对HashMap的一个简单的封装

TreeSet

​ HashSet是无序的，TreeSet实现了SortedSet接口，它是有序的。和TreeMap相同，TreeSet添加的元素必须实现Comparable接口。

Queue

​ 遵循FIFO(first in first out)原则，Queue只能在末尾添加元素，在头部取出元素：

  1. int size()：获得队列的长度
  2. boolean add(E)/boolean offer(E)：添加元素到队尾
  3. E remove()/E poll()：获得队首元素并从队列中删除
  4. E element()/E peek()：获得队首元素但不从队列中删除

  • 不要把null添加到队列中，否则poll()方法无法判断是取得了null还是队列为空

PriorityQueue

​ 队列中的元素具有优先级，调用poll()和peek()等方法时总是返回优先级最高的元素。因此放入优先队列的元素都要实现Comparable接口，如果没有实现该接口，可以提供一个comparator对象来判断两个对象的顺序

  • 面向抽象编程的一个原则是：尽量持有接口，而不是具体的实现类

Stack

​ 栈是一种LIFO(Last In First Out)的数据结构，Stack只有入栈和出栈操作：

  1. push(E)：把元素压栈
  2. pop(E)：把栈顶的元素弹出
  3. peek(E)：取栈顶的元素但不弹出
  • java集合类没有单独的Stack接口，使用Deque实现Stack功能，但要注意禁用一些功能

  1. 不可变集合
  • 封装为不可变集合可以拦截所有修改方法

注解与反射

发表于 2021-01-31 更新于 2021-02-01

注解

  • 注解可以对程序做出解释
  • 可以被其他程序读取
  • 注解还有检查错误的作用

内置注解

  • @Override ：方法重写的注解
  • @Deprecated：代表方法已过时，不推荐使用，有好的替代方法
  • @SuppressWarnings(“all”)：镇压警告需要有参数，镇压全部警告

元注解

​ 负责解释其他注解的注解，有四个类型的元注解：

  1. @Target：用于描述注解的使用范围（可以用在什么地方）

  2. @Retention：表示需要在什么级别保存该注释信息，描述注解的生命周期（SOURCE<CLASS<RUNTIME），一般直接使用RUNTIME

  3. @Documented：说明该注解将被包含在javadoc中

  4. @inherited：说明子类可以继承父类中的注解

自定义注解

​ 使用@interface自定义注解，自动继承java.long.annotation.Annotation接口

  • 注解只有一个参数最好定义参数名为value，使用时可以省略value

反射

  1. 动态语言：在运行时可以改变其结构的语言，例如python，javaScript

  2. 静态语言：通过反射机制可以让java从静态语言编程准动态语言

反射机制允许程序在执行期间获取任何类的内部信息，并能直接操作任意对象的内部属性和方法。

  • 加载类之后，堆内存的方法区就产生了一个Class类型的对象，一个类只有一个Class，我们可以通过这个对象看到类的结构

获得Class类的方法

  1. ClassforName(String name)：获得Class对象
  2. newInstance():返回Class对象的一个实例
  3. getName():返回此Class对象所表示的实体名称
  4. getSuperclass(): 返回当前Class对象的父类Class对象
  5. getinterfaces(): 获取当前Class对象的接口
  6. getClassLoader():返回该类的类加载器
  7. getConstructors():返回一个包含某些Constructor对象的数组
  8. getMethod(String name, Class T):返回一个Method对象。获得本类和父类的所有public方法。加上Declared只能获得本类的所有方法
  9. getDeclaredFields():返回Field对象的一个数组，可以获得private属性。不加Declared只能获得public属性
  • 数组只要数据类型和维度一样，对应的Class类就一样

类的加载

  1. 加载：类到内存，产生Class对象
  2. 链接：验证所有的类都符合规范；为static变量分配内存并使用默认值进行初始化；将常量池中的符号引用转化为直接引用
  3. 初始化，执行将所有静态代码块合并 执行；若父类还未初始化先初始化父类

类的初始化

  • 类的主动引用（一定会发生类的初始化）

    1. new一个对象
    2. 反射
    3. 调用类的静态成员（除了final）和静态方法

    初始化类时，若其父类没有被初始化先初始化父类，虚拟机启动时会先初始化main方法所在的类

  • 类的被动引用（不会发生类的初始化）

    1. 子类调用在父类中定义的静态方法或变量，子类不会初始化
    2. 新建数组不会发生类的初始化
    3. 直接引用常量不会发生类的初始化（在链接阶段已经做了处理）

类加载器

  • 引导类加载器(根加载器)：使用C++编写，无法直接获取
  • 扩展类加载器
  • 系统类加载器
  • 双亲委派机制：多重检测保证安全性。如果自己手写了已经存在包内的类，系统会自动检测每个类加载器是否有相同的类，有相同的则会让手写的类无效。

动态创建对象

  • 在反射中调用时要注意权限问题，权限不足会无法调用方法和属性

  • 权限不足时，可以通过关闭安全检测获取

泛型

发表于 2021-01-31 更新于 2021-02-02

泛型

​ 泛型就是定义了一种模板，可以满足各种类型的需要，避免一些强转类型的情况。

  • ArrayList可以向上转型为List，但不能将ArrayList转为ArrayList

接口中使用泛型

 ArrayLisr.sort(Objec[])可以对任意对象数组进行排序，但是对象必须实现Comparable<T>这个泛型接口

编写泛型

  • 泛型类型不能用于静态方法

檫拭法

  • java编译器使用檫拭法将类型视为Object，然后根据实现安全的强转类型

  • JVM执行的代码并没有泛型

泛型的局限

  1. 不能是基本类型，因为编译器会将T类型视为Object，而Object类型无法持有基本类型

  2. 无法获得带泛型的Class，由于编译时T视为Object，因此不同泛型获得的Class对象都是相同的

  3. 无法判断带泛型的类型

  4. 无法实例化T类型，即不能使用new T()

  5. 不恰当的重写方法

泛型继承

extends通配符

  • Pair不是Pair的子类因此:

  • 使用<? extends Number>定义的类型称为上界通配符，即把泛型T的上界定在Number

  • 方法参数签名<? extends Number>无传递任何Number的子类型

    由于无法传递Number的子类型，编译器并不知道pair对象的泛型为Integer，因此无法固定Integer作为方法的参数

extends通配符作用

  1. 使用extends限制，只能对参数进行读取，无法写（可以写入null）
  2. extends限制T类型

super通配符

  • 使用<? super Integer>表示接收Integer所有父类作为泛型

  • 使用super通配符表示只能堆参数进行写，不能进行读（可以使用Object读）

PECS原则

​ Producer Extends Consumer Super。即生产者使用extends通配符，消费者使用super通配符

无限制通配符

  • ，既不允许写也不允许读，只能做一些null判断
  • Pair<?>是所有Pair的超类

注意点

  • 不能直接创建泛型数组，即不能new Pair[2]，只能强制类型转换，(Pair[])new Pair[2]

  • 在定义泛型时也不能直接创建泛型数组，要使用反射创建，(T[])Array.newInstance(Class, 5)

异常

​ 异常是指程序中出现的不期而至的各种情况：

Java基础（4）

发表于 2021-01-28

面向对象

  • 面向对象的本质：以类的方式组织代码，以对象的组织封装数据
  • 三大特性：封装、继承、多态
  • 类是抽象的，对象是具体的，类是对所有对象的抽象

方法调用

  1. 静态方法：修饰词加了static的方法

  2. 非静态方法：修饰词没有static的方法

  • 静态方法和类一起加载，非静态方法和对象一起加载，静态方法无法直接调用非静态方法

创建对象

  • this 代表一个对象，static方法中不存在this
  • 每个对象创建时都会有默认的初始值
  • 每个类都会有默认的构造器，使用new关键字时会调用构造器

封装

  • 属性私有，提供获得属性的方法
  • 进行封装之后可以规避一些逻辑上的错误（不合法的属性值设置）
  • 可以隐藏代码的实现细节
  • 统一接口

继承

  • 子类会继承父类全部public方法，private无法继承
  • Java中所有类都直接或间接继承Object类
  • Java中只有单继承，没有多继承
  • this代表当前对象，super代表父类对象
  • 构造子类时会默认构造父类，调用父类构造器必须在子类构造器代码第一行
  • super只能在子类的方法或者构造器中
  • 父类的引用可以指向子类
  • 方法重写的名字和参数列表必须相同，修饰符的范围可以变大（public > protected >default > private），抛出的异常范围可以被缩小
  • 静态方法和非静态方法差别很大，静态方法和类相关，非静态方法和对象相关。因此
    1. 静态方法的调用和定义的数据类型相关
    2. 非静态方法的调用和new的对象相关

方法重写只和非静态方法有关

多态

​ 一个对象的实际类型是确定了，指向的引用类型可以不同。

  • 子类的引用和父类的引用可以理解为两个不同的遥控器，由于子类继承了父类，因此子类会比父类更加复杂。如果定义对象为父类类型，他就无法调用父类中没有子类中有的方法。而父类引用调用子类中重写过的方法时，会调用子类的重写的方法。

  • 多态是方法的多态

  • 存在条件：继承关系，方法重写

  • static,private,final方法无法被重写

instanceof 和类型转换

  • instanceof 判断左边的对象是不是右边类的实例，两边没有关系时会编译报错

  • 类型转换：父类转子类要强制转换，子类转父类可以自动转换（可能丢失一些自己的方法）

static关键字

  • 在新建对象时先执行静态代码块，再执行匿名代码块，最后执行构造器。静态代码块只会执行一次

抽象类

  • 抽象类的所有方法都没有具体实现，继承了它的子类需要重写这些方法

  • 无法new抽象类，只是定义了一个模板，一个约束。但抽象类有构造方法

接口

​ 接口只是定义了规范，无法写方法，相当于一个实现的蓝图。

  • 接口可以实现多继承

  • 所有的方法都是public abstract, 所有变量都是public static final

  • 接口无法被实例化

内部类

类的内部再定义一个类

  • 内部类需要通过实例化外部类实例化
  • 内部类可以获得外部类的私有方法和属性
  • 静态内部类只能获得外部类的静态属性
  • 一个java类中可以有多个java类，但只能有一个public 类

方法

​ 方法类似于封装的函数，一般隶属于某个对象，或者某个类。可以在其他地方被引用。

方法的定义

  • 修饰符和传入参数可选
  • 实参和形参：实际调用时传入的参数为实参，定义方法时定义的参数为形参
  • 值传递和引用传递：
    1. 值传递：调用函数时将实际参数复制一份传递到函数中，在函数中对参数进行修改也不会影响实际参数。（java均为值传递)
    2. 引用传递：调用函数时将实际参数的地址传递到函数中，在函数中对参数进行修改会影响实际参数。

方法重载

​ 一个类中有相同的方法名，但有不同的形式参数。

  • 方法名称必须相同
  • 参数列表必须不同（个数不同，类型不同，顺序不同）

命令行传参

​ 传递参数给main()方法，字符串数组

可变参数

  • 每个方法里只能有一个可变参数，且必须放在最后
  • 调用可变参数相当于调用数组

递归

方法内调用方法

  • 递归需要包括递归头和递归体

    1. 递归头：什么时候停止，即什么时候不调用自身方法
    2. 递归体：什么时候调用自身方法

  • 小计算用递归，大计算可能会崩溃

数组

数组的声明和创建

  • 数组中数据编号从0开始

内存分析

  1. 堆：存放new的对象和数组，可以被所有线程共享，不会存放别的对象引用
  2. 栈：存放基本变量类型（包括值），引用对象的变量（对象的数据存放地址）

初始化

  1. 静态初始化

  2. 动态初始化

数组特点

  1. 数组的长度确定，一旦定义长度无法改变
  2. 同一数组中的元素类型必须相同
  3. 元素类型可以是任意类型，包括基本类型和引用类型
  4. 数组下标的合法区间为[0——(length-1)]
  5. 数组也是一个对象

数组的使用

  1. for-each循环，遍历数组

  2. 数组作为参数传入方法中
