发表于 2021-01-30

异常

​ 异常是指程序中出现的不期而至的各种情况：

  1. 检查异常：用户错误或问题，例如文件找不到

  2. 运行异常：逻辑错误，编译时没办法查出来，但可以通过debug避免

  3. 错误：可能和JVM相关，例如栈溢出

  • AWTError为GUI异常

处理异常

  • try catch 一定需要，finally可选
  • 可以有多个catch，捕获多个异常。但是异常的范围要从小到大捕获。
  • throw 可以主动抛出异常，方法抛出异常throws

自定义异常

  • 自定义异常都要继承Exception类

Java基础（4）

发表于 2021-01-28

面向对象

  • 面向对象的本质：以类的方式组织代码，以对象的组织封装数据
  • 三大特性：封装、继承、多态
  • 类是抽象的，对象是具体的，类是对所有对象的抽象

方法调用

  1. 静态方法：修饰词加了static的方法

  2. 非静态方法：修饰词没有static的方法

  • 静态方法和类一起加载，非静态方法和对象一起加载，静态方法无法直接调用非静态方法

创建对象

  • this 代表一个对象，static方法中不存在this
  • 每个对象创建时都会有默认的初始值
  • 每个类都会有默认的构造器，使用new关键字时会调用构造器

封装

  • 属性私有，提供获得属性的方法
  • 进行封装之后可以规避一些逻辑上的错误（不合法的属性值设置）
  • 可以隐藏代码的实现细节
  • 统一接口

继承

  • 子类会继承父类全部public方法，private无法继承
  • Java中所有类都直接或间接继承Object类
  • Java中只有单继承，没有多继承
  • this代表当前对象，super代表父类对象
  • 构造子类时会默认构造父类，调用父类构造器必须在子类构造器代码第一行
  • super只能在子类的方法或者构造器中
  • 父类的引用可以指向子类
  • 方法重写的名字和参数列表必须相同，修饰符的范围可以变大（public > protected >default > private），抛出的异常范围可以被缩小
  • 静态方法和非静态方法差别很大，静态方法和类相关，非静态方法和对象相关。因此
    1. 静态方法的调用和定义的数据类型相关
    2. 非静态方法的调用和new的对象相关

方法重写只和非静态方法有关

多态

​ 一个对象的实际类型是确定了，指向的引用类型可以不同。

  • 子类的引用和父类的引用可以理解为两个不同的遥控器，由于子类继承了父类，因此子类会比父类更加复杂。如果定义对象为父类类型，他就无法调用父类中没有子类中有的方法。而父类引用调用子类中重写过的方法时，会调用子类的重写的方法。

  • 多态是方法的多态

  • 存在条件：继承关系，方法重写

  • static,private,final方法无法被重写

instanceof 和类型转换

  • instanceof 判断左边的对象是不是右边类的实例，两边没有关系时会编译报错

  • 类型转换：父类转子类要强制转换，子类转父类可以自动转换（可能丢失一些自己的方法）

static关键字

  • 在新建对象时先执行静态代码块，再执行匿名代码块，最后执行构造器。静态代码块只会执行一次

抽象类

  • 抽象类的所有方法都没有具体实现，继承了它的子类需要重写这些方法

  • 无法new抽象类，只是定义了一个模板，一个约束。但抽象类有构造方法

接口

​ 接口只是定义了规范，无法写方法，相当于一个实现的蓝图。

  • 接口可以实现多继承

  • 所有的方法都是public abstract, 所有变量都是public static final

  • 接口无法被实例化

内部类

类的内部再定义一个类

  • 内部类需要通过实例化外部类实例化
  • 内部类可以获得外部类的私有方法和属性
  • 静态内部类只能获得外部类的静态属性
  • 一个java类中可以有多个java类，但只能有一个public 类

发表于 2021-01-27

方法

​ 方法类似于封装的函数，一般隶属于某个对象，或者某个类。可以在其他地方被引用。

方法的定义

  • 修饰符和传入参数可选
  • 实参和形参：实际调用时传入的参数为实参，定义方法时定义的参数为形参
  • 值传递和引用传递：
    1. 值传递：调用函数时将实际参数复制一份传递到函数中，在函数中对参数进行修改也不会影响实际参数。（java均为值传递)
    2. 引用传递：调用函数时将实际参数的地址传递到函数中，在函数中对参数进行修改会影响实际参数。

方法重载

​ 一个类中有相同的方法名，但有不同的形式参数。

  • 方法名称必须相同
  • 参数列表必须不同（个数不同，类型不同，顺序不同）

命令行传参

​ 传递参数给main()方法，字符串数组

可变参数

  • 每个方法里只能有一个可变参数，且必须放在最后
  • 调用可变参数相当于调用数组

递归

方法内调用方法

  • 递归需要包括递归头和递归体

    1. 递归头：什么时候停止，即什么时候不调用自身方法
    2. 递归体：什么时候调用自身方法

  • 小计算用递归，大计算可能会崩溃

数组

数组的声明和创建

  • 数组中数据编号从0开始

内存分析

  1. 堆：存放new的对象和数组，可以被所有线程共享，不会存放别的对象引用
  2. 栈：存放基本变量类型（包括值），引用对象的变量（对象的数据存放地址）

初始化

  1. 静态初始化

  2. 动态初始化

数组特点

  1. 数组的长度确定，一旦定义长度无法改变
  2. 同一数组中的元素类型必须相同
  3. 元素类型可以是任意类型，包括基本类型和引用类型
  4. 数组下标的合法区间为[0——(length-1)]
  5. 数组也是一个对象

数组的使用

  1. for-each循环，遍历数组

  2. 数组作为参数传入方法中

多维数组

注释

关键字

Java 中定义的特定的词，用户不能自己随意使用。不需要背诵，慢慢接触。

数据类型

基本类型

数值类型

  1. 整数类型

    • byte: 一个字节，-128~127
    • short: 两个字节，-32768~32767
    • int: 四个字节，-2147483648~2147483647
    • long: 八个字节，-9223372036854775808~9223372036854775808

  2. 浮点类型

    • float: 四个字节
    • double: 八个字节

  3. 字符类型

    • char: 两个字节

布尔类型

boolean类型: 一个字节，true or false

引用类型

类、接口、数组

扩展

整数扩展

  1. 二进制：0b
  2. 八进制：0
  3. 十六进制：0x

浮点数扩展

  • float: 离散、有限，有舍入误差。只能表示大概的值，接近但不等于。一定要避免使用浮点数进行比较！！！！！！！！！。（BigDecimal）

字符扩展

  • 所有的字符本质上都是数字，可以强制类型转换。（和编码有关，例如 Ascall 码，Unicode 编码, \u可以相互转换）

  • 转义字符

类型转换

运算时需要先转换为统一数据类型才可以计算。

强制类型转换

  • (类别)变量名：高内存类型->低内存类型

  • 不能对布尔值转换，不能把对象类型转换为不相干类型

  • 可能会出现内存溢出，或者精度问题

    自动转换

    低内存类型->高内存类型

变量命名规则

  • 所有变量、方法、类名：见名知意

  • 类成员变量：首字母小写，驼峰原则

  • 局部变量：首字母小写，驼峰原则

  • 常量：大写字母和下划线

  • 类名：首字母大写，驼峰原则

  • 方法名：首字母小写，驼峰原则

  • 包名命名：一般使用公司域名倒置，例如 cn.kaixuanwang.xxx

特殊的运算符

  • ++在前先加后赋值，++在后先赋值再加

  • 短路运算：在逻辑运算中，如果前半部分已经可以判断出逻辑结果，后半部分就不会执行

  • **三目运算符” ？：”**：

  • **字符串连接符+**：如果String在末尾，不会将int转化为String。其他情况下均会将int转化为String进行链接
